项目概览

什么是 oh-my-magento？这不仅是一个简单的脚本，而是一个基于 OpenCode 的高级插件，提供了强大的多 Agent 编排系统。通过 11 个独立的 Agent 协同工作，配合 46 个生命周期 Hook、26 个专业工具，以及完整的 Skill/Command/MCP 系统，为 Magento 开发带来前所未有的效率提升。

02

GitHub Copilot 账号管理器

全称：GitHub Copilot Account Manager

基于 @clack/prompts 构建的交互式命令行工具，支持在个人和工作 GitHub Copilot 账号之间快速切换。所有配置安全地存储在本地。 

$ bunx oh-my-magento copilot-x

账号存储: ~/.config/opencode/copilot-x.json (chmod 0600)

鉴权源: ~/.local/share/opencode/auth.json

🔗 登录与导入

  • Add account (OAuth) — GitHub 设备流登录，支持 github.com 及企业版
  • Add account (manual) — 直接粘贴 Token
  • Import from auth.json — 从 OpenCode 自动检测并导入

⚙️ 管理与诊断

  • Check models — 查看跨账号的可用和被禁用的模型
  • Refresh identity — 更新所有账号的用户名和组织信息
  • Switch account — 一键切换激活的 Copilot 账号

🗑️ 账号清理

  • Remove account — 移除指定的单个账号
  • Remove all accounts — 破坏性操作：移除所有存储的账号
03

Token 使用与效能分析

全称：Token Usage Statistics & Analytics

# 基础使用
$ bunx oh-my-magento usage --days 30 --by model

# 时间过滤与环比对比
$ bunx oh-my-magento usage --since 7d --compare

# 生成并保存日报表
$ bunx oh-my-magento usage report --date 2026-03-14

关键指标 (需 dev-metrics 支持)

  • 每次提交成本 (Cost per commit)
  • 每行代码 Token (Tokens per LOC)
  • 输出密度 (Output density)
  • 会话生产力得分 (Session productivity score)

📊 核心参数

  • --days <n> 显示过去 N 天的数据
  • --since <spec> 时间过滤 (7d, 2w, 3h 或 ISO 日期)
  • --by <dim> 维度分组：day, model, agent, provider, session, project 等
  • --limit <n> 限制显示的行数
  • --json 输出 JSON 格式数据
  • --compare 与上一周期的同等时间段进行对比

📈 显示特性

  • 带微型柱状图的每日数据明细
  • 详尽的 Token 分解：输入、输出、缓存读取、缓存写入
  • 精准到每次调用、每个模型及 Agent 的成本追踪
  • 带增量百分比的环比数据分析

其他实用命令

oh-my-magento 提供的完整工具链，覆盖安装、诊断与运行生命周期。

⚡

install

交互式安装向导，自动配置 OpenCode 插件及环境变量，一键就绪。

🩺

doctor

全方位健康诊断工具，检查系统环境、配置文件、可用工具和模型状态。

🚀

run <message>

非交互式会话启动器，支持直接通过命令行传递任务指令，适合集成到自动化流程中。

🏷️

get-local-version

快速版本检查工具，验证当前安装的插件版本是否为最新。

🔑

mcp-oauth

OAuth 令牌管理器，专用于配置和管理外部 MCP (Model Context Protocol) 服务的身份验证。
